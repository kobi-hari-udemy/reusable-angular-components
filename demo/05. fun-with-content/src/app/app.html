What is Content Projection?

The Core Idea

Content projection is a pattern in Angular that allows a component to accept and render markup provided by its parent. Rather than hardcoding its inner structure, a component defines slots where external content is inserted at runtime.

How It Works

Using the ng-content element, a component declares one or more projection slots. The parent then passes arbitrary HTML or even other components into those slots, keeping the child component flexible, reusable, and decoupled from the content it displays.

Creating Customizable Components

Beyond Inputs

Instead of relying on inputs for every visual variation, a component can expose projection slots that let consumers inject their own templates, icons, or layouts. This keeps the component API clean while offering unlimited flexibility.

Real-World Benefits

Composite Components

Composition Over Configuration

Composite components take reusability further by combining multiple smaller, focused components into a cohesive whole. Each sub-component handles a single concern and communicates through well-defined interfaces.

Building Blocks

Think of a data table built from separate column, header, and row components, or a form layout assembled from field wrappers and validation indicators. This pattern lets consumers mix and match building blocks to create highly customizable UIs without modifying internal component code.
